Overview

AquaLink is a lightweight and efficient Lavalink wrapper designed for performance and stability. It allows developers to integrate Lavalink into their Discord bots with minimal dependencies. AquaLink handles voice connections, track playback, and queue management with ease.

Features

  • Minimal Dependencies
  • Built-in Queue Management
  • Auto-Reconnect for Lavalink Nodes
  • High-Quality Audio Streaming
  • Supports Multiple Lavalink Nodes
  • Easy-to-Use API

Installation

To install Aqualink, ensure you have Node.js 16.9.0 or higher and npm installed, then run:

Terminal 
npm install aqualink

You'll also need a Lavalink server running. For instructions on setting up Lavalink, visit the Lavalink GitHub repository.

Configuration

Configure your Aqualink instance with the following options:

JavaScript 
const { Client, GatewayIntentBits } = require('discord.js');
const { Aqua } = require('aqualink');

const client = new Client({
  intents: [
    GatewayIntentBits.Guilds,
    GatewayIntentBits.GuildVoiceStates
  ]
});

// Create Aqua instance with configuration
const aqua = new Aqua(client, [
  {
    name: "main-node",     // Optional but recommended for identification
    host: "localhost",     // Lavalink server address
    port: 2333,            // Lavalink server port
    password: "youshallnotpass", // Lavalink server password
    secure: false          // Whether to use WSS (secure websocket)
  }
], {
  // Optional configuration
  defaultSearchPlatform: 'ytsearch', // Default platform for searches
  shouldDeleteMessage: false,     // Whether to delete messages after processing
  leaveOnEnd: true,               // Leave voice channel when playback ends
  autoResume: false,              // Automatically resume disconnected sessions
  infiniteReconnects: false       // Whether to infinitely attempt reconnections
});

Basic Usage

Here's how to initialize Aqualink and start playing music:

JavaScript 
// Initialize Aqualink when your Discord client is ready
client.once('ready', () => {
  // Pass your client ID to initialize
  aqua.init(client.user.id);
  console.log('Aqualink initialized!');
});

// Handle voice state updates from Discord
client.on('raw', (packet) => {
  if (packet.t === 'VOICE_SERVER_UPDATE' || packet.t === 'VOICE_STATE_UPDATE') {
    aqua.updateVoiceState(packet);
  }
});

// Example slash command to play music
async function playCommand(interaction) {
  const query = interaction.options.getString('song');
  const voiceChannel = interaction.member.voice.channel;
  
  if (!voiceChannel) {
    return interaction.reply('You need to be in a voice channel!');
  }
  
  // Get or create a player for this guild
  let player;
  try {
    player = aqua.get(interaction.guildId);
  } catch {
    player = aqua.createConnection({
      guildId: interaction.guildId,
      voiceChannel: voiceChannel.id,
      textChannel: interaction.channelId
    });
  }
  
  // Search for the track
  const result = await aqua.resolve({
    query,
    requester: interaction.user
  });
  
  if (result.loadType === 'error' || !result.tracks.length) {
    return interaction.reply('No results found!');
  }
  
  // Add track to queue and play if not already playing
  player.queue.add(result.tracks[0]);
  
  if (!player.playing) {
    player.play();
  }
  
  return interaction.reply(`Added to queue: ${result.tracks[0].title}`);
}

Queue System

Aqualink comes with a powerful queue management system:

JavaScript 
// Get the player for a guild
const player = aqua.get(guildId);

// Access the queue
const queue = player.queue;

// Add a track to the queue
queue.add(track);

// Add multiple tracks
queue.add(tracks);

// Get current track
const currentTrack = queue.current;

// See upcoming tracks
const upcomingTracks = queue.tracks;

// Skip to the next track
player.skip();

// Shuffle the queue
player.shuffle();

// Set loop mode (none, track, queue)
player.setLoop('queue');

// Clear the queue but keep the current track
queue.clear();

// Remove a specific track by index
queue.remove(2); // Removes the third track in the queue

Player API

The Player class provides rich functionality for audio playback control:

Playback Control Methods

play()

Plays the next track in the queue.

pause(paused)

Toggles the pause state of the player.

Parameters:

  • paused (Boolean): Whether to pause or unpause

stop()

Stops the current track and resets position.

seek(position)

Seeks to a position in the current track.

Parameters:

  • position (Number): Position in milliseconds to seek to

setVolume(volume)

Sets the playback volume.

Parameters:

  • volume (Number): Volume level (0-200)

skip()

Skips the current track and plays the next one.

Queue Management Methods

shuffle()

Shuffles the tracks in the queue.

setLoop(mode)

Sets the loop mode.

Parameters:

  • mode (String): Loop mode ("none", "track", or "queue")

setAutoplay(enabled)

Enables or disables autoplay functionality.

Parameters:

  • enabled (Boolean): Whether to enable or disable autoplay

Aqua Node Documentation

Overview

In Aqua, there are two primary ways to access node information:

  1. aqua.nodes: An array containing node configurations
  2. aqua.nodeMap: A Map object containing instantiated Node class instances

Node Class

The Node class establishes and maintains connections to Lavalink servers.

Constructor Options

JavaScript 
const node = new Node(aqua, {
                name: "primary-node",       // Optional, defaults to host
                host: "localhost",          // Required
                port: 2333,                 // Optional, defaults to 2333
                password: "youshallnotpass", // Optional, defaults to "youshallnotpass"
                secure: false,              // Optional, enables WSS/HTTPS
                regions: ["us", "eu"],      // Optional, for region-specific routing
                sessionId: null             // Optional, for resuming connections
            }, {
                resumeTimeout: 60,          // Optional, seconds to keep session alive
                autoResume: false,          // Optional, auto-resume disconnected sessions
                reconnectTimeout: 2000,     // Optional, base reconnect delay (ms)
                reconnectTries: 3,          // Optional, max reconnection attempts
                infiniteReconnects: false   // Optional, keep trying to reconnect indefinitely
            });

Properties

Connection Properties

  • name: Identifier for the node (defaults to host)
  • host: Hostname or IP address of the Lavalink server
  • port: Port number of the Lavalink server
  • password: Password for Lavalink authentication
  • secure: Whether to use secure WebSocket/HTTP connections
  • sessionId: Session ID for resuming disconnected sessions
  • wsUrl: WebSocket URL for Lavalink connections
  • connected: Whether the node is currently connected
  • regions: Array of region identifiers for region-specific routing

Configuration Properties

  • resumeTimeout: Time in seconds to keep session alive for resuming
  • autoResume: Whether to automatically resume disconnected sessions
  • reconnectTimeout: Base delay in milliseconds between reconnection attempts
  • reconnectTries: Maximum number of reconnection attempts
  • infiniteReconnects: Whether to continue reconnection attempts indefinitely

Node Load Balancing

JavaScript 
function getBestNode(guild) {
                // Get available nodes
                const nodes = Array.from(aqua.nodeMap.values()).filter(node => node.connected);
                if (!nodes.length) return null;
                
                // If guild has a region, try to find a region-specific node
                if (guild.region) {
                    const regionNodes = nodes.filter(node => 
                        node.regions.includes(guild.region.toLowerCase())
                    );
                    if (regionNodes.length) {
                        // Return the least used node in the region
                        return regionNodes.sort((a, b) => 
                            (a.stats.players / a.stats.cpu.cores) - 
                            (b.stats.players / b.stats.cpu.cores)
                        )[0];
                    }
                }
                
                // Otherwise return the least used node overall
                return nodes.sort((a, b) => 
                    (a.stats.players / a.stats.cpu.cores) - 
                    (b.stats.players / b.stats.cpu.cores)
                )[0];
            }

            // Usage
            const bestNode = getBestNode(interaction.guild);
            const player = aqua.createPlayer(bestNode, {
                guildId: interaction.guildId,
                voiceChannel: interaction.member.voice.channel.id
            });

Monitoring Node Health

JavaScript 
// Set up interval to check node health
            setInterval(() => {
                const nodes = Array.from(aqua.nodeMap.values());
                
                // Log node statistics
                console.log("=== Node Status ===");
                nodes.forEach(node => {
                    const { stats } = node;
                    console.log(`${node.name}: ${stats.players} players, CPU: ${stats.cpu.lavalinkLoadPercentage.toFixed(1)}%, Memory: ${(stats.memory.used / 1024 / 1024).toFixed(0)}MB`);
                });
                
                // Alert on high load
                const overloadedNodes = nodes.filter(node => 
                    node.stats.cpu.lavalinkLoadPercentage > 80
                );
                
                if (overloadedNodes.length) {
                    console.warn(`Warning: ${overloadedNodes.length} nodes are experiencing high load!`);
                }
            }, 60000);

REST API

Available Endpoints

  • getRoutePlannerAddress(): Retrieves the address of the route planner.
  • getRoutePlannerStatus(): Gets the current status of the route planner.
  • getInfo(): Fetches general information about the system or service.
  • getStats(): Retrieves statistical data regarding usage or performance.
  • decodeTracks(): Decodes multiple tracks based on provided data.
  • decodeTrack(): Decodes a single track based on provided data.
  • getTracks(): Fetches a list of available tracks.
  • destroyPlayer(): Stops and destroys the specified player instance.
  • getPlayers(): Retrieves a list of active players in the system.
  • updatePlayer(): Updates the settings or status of a specified player.
  • makeRequest(): Sends a generic request to the server.
  • setSessionId(): Sets the session ID for user interaction with the system.

Track API

Track Properties

  • identifier: string; // Unique identifier for the track
  • isSeekable: boolean; // Indicates if the track can be seeked
  • author: string; // Author or artist of the track
  • length: number; // Length of the track in milliseconds
  • isStream: boolean; // Indicates if the track is a live stream
  • position: number; // Current playback position in the track
  • title: string; // Title of the track
  • uri: string; // URI of the track
  • sourceName: string; // Name of the source (e.g., platform)
  • artworkUrl: string; // URL of the track's thumbnail image
  • track: string; // A specific track's identifier
  • tracks: Array<Track>; // Array of related tracks
  • playlist:
    • name: string; // Name of the playlist
    • selectedTrack: number; // Index of the currently selected track in the playlist

            player.setFilter({

            // Reset filters
            player.clearFilter();

FAQ

Q: What are the supported music sources?

Aqualink supports these music sources:

  • YouTube (ytsearch)
  • YouTube Music (ytmsearch)
  • Spotify (spsearch)
  • SoundCloud (scsearch)
  • Deezer (dzsearch)
  • Tidal (tdsearch)
  • Apple Music (amsearch)
  • Bandcamp (bcsearch)
  • JioSaavn (jssearch)

Q: How do I handle voice disconnections?

You can enable the autoResume option when creating your Aqua instance, or handle reconnections manually by listening to the socketClosed event.

Q: Can I use multiple Lavalink nodes?

Yes! Aqualink supports multiple nodes with automatic load balancing. Simply provide an array of node configurations when creating your Aqua instance.

Q: How do I enable debugging?

you can use the event emitter: aqua.on('debug', (message) => console.log(message));
